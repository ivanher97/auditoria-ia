El problema y la solución

Pasar currículums recibidos en cualquier formato a la plantilla corporativa es un trabajo manual, lento y propenso a errores de copia.

😓 Antes

  • Leer cada PDF y transcribir experiencia, formación y habilidades a mano.
  • Copiar y pegar en la plantilla de Word cuidando el formato.
  • Errores de transcripción y estilos rotos difíciles de detectar.
  • Minutos —a veces horas— por cada candidato.

✨ Ahora

  • Se selecciona el PDF y la aplicación extrae el texto al instante.
  • La persona revisa ese texto (y borra lo que no deba salir del equipo).
  • Un clic: la IA organiza la información y rellena la plantilla corporativa.
  • El Word queda guardado junto al PDF original, listo para retocar.

Preguntas frecuentes

El archivo PDF nunca sale del ordenador. Lo único que se envía a la IA es el texto que la persona ya ha revisado en pantalla — y puede borrar de él cualquier dato antes de continuar.

La respuesta de la IA se comprueba automáticamente contra una estructura obligatoria antes de generar nada; si no encaja, la aplicación avisa. Además, el resultado es un Word normal: siempre se puede retocar a mano.

Decisiones técnicas y su porqué

Fiabilidad

JSON Schema como contrato con el LLM

La salida del modelo se valida contra un esquema estricto antes de tocar el Word. Un modelo probabilístico se convierte así en una pieza determinista: o la respuesta cumple el contrato, o falla de forma explícita — nunca un documento corrupto en silencio.

Privacidad y coste

Extracción de texto 100 % local

El PDF se procesa en el equipo con pypdf y limpieza de artefactos OCR. A la API solo viaja texto plano ya revisado: menos superficie de exposición de datos personales y menos tokens facturados.

Cumplimiento

Human-in-the-loop antes de la API

El paso de revisión no es cosmético: es el punto de control donde se eliminan datos sensibles antes de salir del equipo. La decisión de qué ve la IA es siempre humana.

Mantenibilidad

Plantillas docxtpl separadas del código

La presentación vive en la plantilla .docx; el código solo inyecta datos. Cambiar la imagen corporativa no requiere tocar una línea de Python.

Coste / latencia

Modelos Flash seleccionables

Gemini 2.5 Flash, 3 Flash (preview) y 3.1 Flash Lite, elegibles desde la interfaz: permite ajustar el equilibrio calidad-coste-velocidad según el lote de CVs.

Estado del proyecto

Prototipo funcional en uso

Es la más ligera de las tres aplicaciones del portfolio (~700 líneas): un pipeline enfocado que hace una cosa y la hace bien. Se distribuye como ejecutable autocontenido y ha pasado por varias iteraciones de versión (builds V1.3 y V1.3.1 empaquetadas).

Evolución natural: añadir una suite de tests automatizados del pipeline (hoy no la tiene), endurecer el manejo de errores de la API y procesar lotes de CVs en una sola pasada.
